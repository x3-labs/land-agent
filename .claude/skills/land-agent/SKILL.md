# Land Agent — подключение сервисов к лендингу

Этот скилл запускается после того как лендинг уже создан.
Пользователь — маркетолог или владелец бизнеса без технических знаний. Объясняй просто, без жаргона.

## Порядок работы

1. Задать вопросы Блока 1 (все три сразу)
2. Дождаться ответов на все три
3. Задать вопросы Блока 2 (все три сразу)
4. Дождаться ответов
5. Внести все изменения в файлы за один проход
6. Вывести итоговый чеклист

Не вносить никаких изменений в файлы до получения ответов на все вопросы Блока 1.

---

## Блок 1 — Обязательные вопросы

Задать все три вопроса **одним сообщением**. Не переходить к Блоку 2 пока не получены ответы на все три.

### Как задавать

```
Прежде чем подключать сервисы, мне нужна информация. Ответьте на три вопроса:

**1. Webhook для приёма заявок**
Укажите адрес webhook — это адрес из n8n, Make или другого сервиса автоматизации.
Выглядит так: https://hook.make.com/xxx

**2. GTM ID для аналитики**
Укажите ваш GTM ID.
Выглядит так: GTM-XXXXXXX
Найти можно в личном кабинете на tagmanager.google.com

**3. Названия форм**
[Здесь вставить результат анализа форм — см. ниже]
```

### Анализ форм перед вопросом 3

До отправки сообщения самостоятельно прочитать index.html и найти все теги `<form>`.

Для каждой формы:
- Взять значение атрибута `id` или `name` если есть
- Если нет — присвоить порядковый номер: `form-1`, `form-2` и т.д.
- Предложить человекочитаемое название на русском исходя из контекста формы (где она стоит, какие поля)

Примеры человекочитаемых названий: "Главная форма", "Попап", "Форма в конце страницы", "Форма записи".

Вставить в вопрос 3:
```
Нашёл [N] форм на странице:
— [selector формы 1] → предлагаю назвать "[Название 1]", ок?
— [selector формы 2] → предлагаю назвать "[Название 2]", ок?
Подтвердите или напишите свои названия.
```

### Объяснения если пользователь не знает

**Не знает что такое webhook:**
> "Webhook — это адрес куда будут приходить заявки с форм. Его нужно создать в вашем сервисе автоматизации (n8n, Make и др.) и вставить сюда."

**Не знает что такое GTM:**
> "Google Tag Manager — сервис для подключения аналитики и пикселей без правки кода. GTM ID есть в вашем аккаунте на tagmanager.google.com — выглядит как GTM-XXXXXXX."

## Что делать после получения всех ответов

Внести все изменения за один проход. Не редактировать файл несколько раз подряд.

### Всегда — hook-form.js

Добавить перед `</body>` в index.html:

```html
<!-- land-agent: hook-form -->
<script src="https://cdn.jsdelivr.net/gh/x3-labs/hook-form@v1.0.0/hook-form.js"></script>
<script>
  HookForm.init({
    webhook: 'WEBHOOK_URL'
  });
</script>
```

Заменить `WEBHOOK_URL` на указанный пользователем адрес.

Если в `HookForm.init()` нужен редирект — добавить параметр `redirect` (см. ниже).

### Всегда — атрибуты name у форм

Проверить каждый `<form>` в index.html. Если у формы нет атрибута `name` — добавить согласно подтверждённым названиям из Блока 1. Транслитерировать или использовать латинские короткие имена (например "hero-form", "popup-form").

### Всегда — GTM

Добавить в `<head>` стандартный GTM-сниппет с реальным GTM ID пользователя:

```html
<!-- land-agent: GTM -->
<!-- Google Tag Manager -->
<script>(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':
new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],
j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src=
'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);
})(window,document,'script','dataLayer','GTM-XXXXXXX');</script>
<!-- End Google Tag Manager -->
```

Добавить GTM noscript сразу после открывающего `<body>`:

```html
<!-- land-agent: GTM noscript -->
<!-- Google Tag Manager (noscript) -->
<noscript><iframe src="https://www.googletagmanager.com/ns.html?id=GTM-XXXXXXX"
height="0" width="0" style="display:none;visibility:hidden"></iframe></noscript>
<!-- End Google Tag Manager (noscript) -->
```

Заменить оба `GTM-XXXXXXX` на реальный GTM ID.

### По ответам Блока 2

**Страница спасибо (вариант а):**

Создать `thank-you.html` в той же папке что `index.html`.

Страница должна:
- Использовать те же CSS-переменные, шрифты и цвета что в index.html
- Содержать сообщение благодарности и что будет дальше
- Быть адаптивной (mobile-first)
- Иметь ссылку "Вернуться на главную" → `index.html`

В `HookForm.init()` добавить параметр редиректа:
```js
HookForm.init({
  webhook: 'WEBHOOK_URL',
  redirect: '/thank-you.html'
});
```

**Редирект на внешний сервис (вариант б):**

В `HookForm.init()` добавить параметр редиректа с указанной ссылкой:
```js
HookForm.init({
  webhook: 'WEBHOOK_URL',
  redirect: 'https://указанная-ссылка.com'
});
```
`thank-you.html` не создавать.

**Политика конфиденциальности:**

Добавить ссылку в футер `index.html` (и `thank-you.html` если создан):
```html
<a href="ССЫЛКА" target="_blank" rel="noopener">Политика конфиденциальности</a>
```

**Договор оферты:**

Добавить ссылку в футер `index.html` (и `thank-you.html` если создан):
```html
<a href="ССЫЛКА" target="_blank" rel="noopener">Договор оферты</a>
```

Если в футере уже есть другие ссылки — добавить рядом, не ломая вёрстку.

## Итоговый чеклист

После внесения всех изменений вывести пользователю:

```
✅ Готово! Вот что подключено:

— Отправка форм → [webhook URL]
— GTM → [GTM ID]
— Формы: [список: "Название 1", "Название 2"]
[— Страница спасибо: thank-you.html создана]
[— Редирект после формы: ссылка]
[— Политика конфиденциальности: добавлена в подвал]
[— Договор оферты: добавлен в подвал]

Проверьте лендинг в браузере и отправьте тестовую заявку.
```

Пункты в `[квадратных скобках]` показывать **только** если соответствующая настройка была выбрана.
